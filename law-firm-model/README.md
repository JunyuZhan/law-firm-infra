# 律师事务所管理系统 - 数据模型层

## 模块说明

数据模型层定义了系统所有的数据结构和业务模型，采用DDD（领域驱动设计）思想，将业务模型按照领域进行划分。每个模型模块都包含实体（Entity）、数据传输对象（DTO）、视图对象（VO）等。

## 模块结构

### 1. 基础模型 (base-model)
系统基础模型定义，为其他模型提供基础支持：
- entity: 基础实体类
- dto: 数据传输对象
- vo: 视图对象
- query: 查询对象
- result: 统一返回结果
- enums: 枚举定义
- constants: 常量定义
- support: 工具支持类
- tenant: 租户相关模型
- geo: 地理位置模型
- service: 基础服务接口

## 开发规范

### 1. 命名规范
- 实体类：XxxEntity
- DTO类：XxxDTO
- VO类：XxxVO
- 查询类：XxxQuery
- 枚举类：XxxEnum
- 常量类：XxxConstants

### 2. 模型规范
- 所有实体类都需要继承 BaseModel
- DTO类需要实现序列化接口
- 查询类需要继承 BaseQuery
- 必须提供合适的构造方法
- 使用 Lombok 简化代码
- 字段必须添加注释

### 3. 验证规范
- 使用 JSR-303 注解进行参数验证
- 自定义验证注解需要添加文档
- 验证分组需要合理划分
- 错误信息需要国际化

### 4. 数据库规范
- 表名使用下划线命名法
- 字段名使用下划线命名法
- 必须包含基础字段（id, created_time, updated_time等）
- 需要添加合适的索引
- 大字段需要单独建表
