# 接口说明文档

## 接口规范

所有接口遵循RESTful风格设计，返回统一的数据格式：

```json
{
  "code": 200,  // 状态码，200表示成功，非200表示失败
  "message": "操作成功",  // 提示信息
  "data": {}  // 返回的数据，可能是对象、数组或null
}
```

## 常见状态码

- 200: 请求成功
- 400: 请求参数错误
- 401: 未授权或授权已过期
- 403: 权限不足
- 404: 资源不存在
- 500: 服务器内部错误

## 认证方式

系统使用JWT进行接口认证，步骤如下：

1. 调用登录接口获取Token
2. 在请求头中添加 `Authorization: Bearer {token}`

## 接口分组

系统接口分为以下几个模块：

- 认证接口: 处理用户登录、注册等
- 系统管理: 用户、角色、权限等基础配置
- 业务管理: 案件、客户、合同等核心业务
